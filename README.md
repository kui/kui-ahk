# kui-ahk

AutoHotkey v2 による個人用キーボードカスタマイズスクリプト。

## 概要

- **F18 をモディファイアキーとして活用**し、ホームポジションを崩さずに各種操作を行う
- **IME の状態を視覚的にフィードバック**し、日本語・英数入力の切り替えを快適にする

## 要件

- [AutoHotkey v2](https://www.autohotkey.com/)
- Windows 10 / 11

## 使い方

`main.ahk` をダブルクリック、またはスタートアップに登録して実行する。

---

## F18 モディファイアキー

F18 を**押しながら**別のキーを押すことで、ホームポジションを維持したまま様々な操作を行う「第2レイヤー」として機能する。

### 設計思想

通常のモディファイアキー（Ctrl・Alt・Win）と組み合わせることでバリエーションを広げている。

| 組み合わせ | 用途の方向性 |
|---|---|
| F18 + キー | 基本操作（カーソル移動・編集・IME 等） |
| F18 + Ctrl + キー | 拡張カーソル操作（単語単位・ページ単位） |
| F18 + Alt + キー | 大きな移動（行頭/末・ドキュメント先頭/末） |
| F18 + Shift + キー | 拡張操作（Redo 等） |

### 未定義キーの無効化

F18 を押している間、**バインドが定義されていないキーはすべて無効化される**。F18 押下中に誤って意図しない文字が入力されることを防ぐための設計。

## IME 制御と表示

### 状態の監視

IME の状態変化はポーリングで検出している。イベント駆動が理想だが、アクティブウィンドウ・フォーカスコンポーネント・IME ウィンドウを連鎖してフックするのは複雑なため、ポーリングを採用している。

### 状態変化の通知（センターオーバーレイ）

IME の状態が切り替わったとき、接続されている**全モニターの中央**にインジケーターが一時的に表示される。複数モニター環境でも、どのモニターを見ていても気づけるよう全画面に表示する。

### マウスカーソル付近のインジケーター

日本語入力モード中は、マウスカーソルの近くに小さなインジケーターが常時表示される。センターオーバーレイが消えた後も現在の入力モードを確認できる。

### マウス移動による自動 IME 切り替え

日本語入力モード中に、マウスカーソルが一定距離以上移動すると、自動的に**英数入力モードへ切り替わる**。

マウスに手が移ったということは「文字を打つフェーズが終わった」とみなせるため、IME をリセットしておくことで誤入力を防ぐ。
